# Upgrade: agent-index-marketplace v1.x → v2.0.0

## Summary

v2.0.0 migrates all marketplace operations from the shared-mount-drive model to the remote filesystem model. All org-level reads/writes now use `aifs_*` remote filesystem tools. Requires agent-index-core v2.0.0.

---

## Breaking Changes

1. **All tasks now use `aifs_*` tools**: `download-collection`, `install-collection`, `list-marketplace-collections`, `list-org-collections`, `refresh-marketplace-cache`, and `check-updates` all read/write via the remote filesystem tools instead of direct file access.
2. **`download-collection` no longer supports git clone**: Collections are downloaded as ZIP and uploaded to the remote filesystem via `aifs_write`.
3. **Marketplace directory is network-only**: The bundled `marketplace-directory.json` was removed in v1.0.1. First-time setup and cache misses require network access.
4. **`@ai:fs-setup` references replaced**: All references to `@ai:fs-setup` now point to `@ai:member-bootstrap`.

---

## Migration Steps

### Step 1 — Prerequisite

Ensure agent-index-core has been upgraded to v3.0.0 first (the on-demand executor model). The marketplace v2.0.0 depends on the remote filesystem infrastructure that core provides. See `agent-index-core/upgrade/2-to-3.md` for the core migration guide.

### Step 2 — Admin: Update Collection Files

Run `@ai:publish-updates` after upgrading the marketplace collection on the remote filesystem. This generates a `collection-update` entry for the marketplace.

### Step 3 — Members: Apply Update

Members run `@ai:apply-updates`, which will:

1. Update local marketplace task definitions to v2.0.0
2. Re-run setup for any marketplace tasks that have changed parameters
3. Clear and rebuild the local marketplace cache (the cache format changed)

---

## Preserved Member Data

- Marketplace cache will be rebuilt automatically on next use
- No member-specific configuration is lost — the marketplace tasks have no member-defined parameters

---

## Notes

- Members must have a working remote filesystem connection (`aifs_auth_status()` returns `authenticated: true`) before marketplace operations will work in v2.0.0
- The `check-updates` task was added in v2.0.0 and will be offered to members as a new capability during the update flow
